The association class LC_InterStrataRelationship defines the relationship between instances of the LC_Stratum metaobject. <br/>A constraint is that {An instance of LC_Stratum metaclass many not have an interStrataRelationship with itself}<br/>The association class LC_InterStrataRelationship has two attributes <i>presenceType </i>and<i> onTop</i>. The permitted values of the attribute <i>presenceType </i>are described by the enumerated value class LC_StratumPresenceType. The permitted values of the attribute <i>onTop </i><font color="#0000ff">defined by the enumerated value class LC_OnTopType</font>. <br/>The attribute <i>presenceType </i>allows one to indicate whether a stratum is fixed (mandatory); that is, it is always present in the composition of elements for an LC_LandCover metaobject, or whether it is conditional. This allows one to model a situation where a certain stratum is permitted but not always present in the definition of an LC_LandCover metaobject. For example, in describing a type of forested area the model, and describe that a stratum containing the metaobject representing a tree is fixed (mandatory) whereas one for a shrub is conditional (optional). The <i>presenceType</i> of conditional is useful when describing a LCCS that contains classes or legends based on ambiguous definitions or when one wants to indicate that a further refinement of the land cover class is possible.<br/>The <i>onTop</i> attribute indicates a relationship between strata where one stratum is defined in reference to <font color="#400040">another. For example three elements trees, shrubs and herbs might exist in separate stratum and all originate from the same soil background. See Annex C.3 for an example using three strata. By default if the <i>onTop</i> relationship flag is not specified then all the strata within an LC_LandCover metaobject have a common background reference. That is, the default relationship is to the common baseline. Other values of the <i>onTop</i> relationship must be defined explicitly. An example a "roof tree garden" could be expressed as a building forming the first stratum with the object tree forming the second stratum on top of the first stratum. This construct is very useful to describe trees with epiphytes or lianas, boulders with lichen or mosses etc.</font><br/><font color="#400040">There are twoconstraints on the <i>onTop</i> relationship:.</font><br/><font color="#400040">{If two or more LC_Element in the stratum are regulated by a exclusive or temporal relationship the element of the stratum with an <i>onTop</i> relation goes "on top" of all of the elements regulated by the exclusive or temporal relationship.}  and  </font><br/><font color="#400040">{If two or more LC_Elements are regulated by a "conditional" relationship the LC_Element from the on top stratum goes "on top" of the "fixed" element in the conditional relationship.}</font><br/><br/>
